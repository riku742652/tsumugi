## Zaim CSV エクスポートフォーマット

Zaim プレミアムの「設定 → ファイル入出力 → ダウンロード」から取得できる CSV。
文字コード: UTF-8（BOMなし）、区切り: カンマ。

### カラム一覧

| # | カラム名 | 型 | 説明 |
|---|---------|-----|------|
| 1 | 日付 | YYYY-MM-DD | 取引日 |
| 2 | 方法 | 文字列 | `支出` / `収入` / `振替` |
| 3 | カテゴリ | 文字列 | 大カテゴリ（食費、交通費 など） |
| 4 | カテゴリの内訳 | 文字列 | 小カテゴリ（外食、電車 など） |
| 5 | 支払元 | 文字列 | 支出元の口座・財布名 |
| 6 | 入金先 | 文字列 | 収入・振替先の口座名 |
| 7 | 品目 | 文字列 | 商品名・品目（任意入力） |
| 8 | メモ | 文字列 | 自由記述メモ |
| 9 | お店 | 文字列 | 店舗名 |
| 10 | 通貨 | 文字列 | `JPY` など |
| 11 | 収入 | 数値 | 収入の場合の金額（支出・振替は空） |
| 12 | 支出 | 数値 | 支出の場合の金額（収入・振替は空） |
| 13 | 振替 | 数値 | 振替の場合の金額（支出・収入は空） |

### サンプル行

```csv
日付,方法,カテゴリ,カテゴリの内訳,支払元,入金先,品目,メモ,お店,通貨,収入,支出,振替
2024-03-15,支出,食費,外食,財布,,ランチ,,松屋,JPY,,850,
2024-03-15,収入,給与,給与,,,3月分給与,,,JPY,280000,,
2024-03-20,振替,,,財布,銀行口座,,,ATM,JPY,,,20000
```

### 注意点

- `収入` / `支出` / `振替` の3カラムは排他的（1行に1つだけ値が入る）
- `計算対象` カラムが存在する場合がある（0=家計に含めない、1=含める）
- カテゴリが空の行（未分類）が存在しうる
- 同日に複数行存在する

## 技術選定候補

### フロントエンド

| 選択肢 | メリット | デメリット |
|--------|---------|----------|
| Vite + TypeScript | 型安全、開発体験良好、ビルド後は静的ファイル | Node.js 環境が必要 |
| 素の HTML + JS | ビルド不要、ファイルを開くだけで動く | 型なし、大規模化に弱い |

**推奨: Vite + TypeScript** — CSV の型定義が書けるため安全。

### CSV パース

| ライブラリ | 特徴 |
|----------|------|
| **PapaParse** | ブラウザ対応、日本語OK、型付き、シェア最大 |
| csv-parse | Node.js 向け、ブラウザ利用には工夫が要る |

**推奨: PapaParse**

### グラフ描画

| ライブラリ | 特徴 |
|----------|------|
| **Chart.js** | シンプルAPI、軽量、棒・折れ線・円すべて対応 |
| Recharts | React 専用、宣言的だが重い |
| D3.js | 高自由度だが学習コストが高い |

**推奨: Chart.js**

## 表示したいグラフ（案）

1. **月別 支出合計** — 棒グラフ（月ごとに支出の推移を把握）
2. **カテゴリ別 支出内訳** — ドーナツグラフ（選択月のカテゴリ比率）
3. **収入 vs 支出** — 積み上げ棒グラフ（月ごとの収支バランス）
4. **カテゴリ別 月次推移** — 折れ線グラフ（特定カテゴリのトレンド）

---

## 制約・注意点

- ファイルはブラウザ内のみで処理（サーバー送信なし）— プライバシー上必須
- Zaim プレミアム限定機能のため、CSV フォーマットが将来変わる可能性がある
- 大量データ（数千行）でも動作するよう、描画は非同期で行う
